Git指定用户进行Clone

Posted on 2018-01-04 | Edited on 2020-03-04

背景

日常开发中，大部分情况下我们都是把本机的ssh公钥，放到git的ssh keys设置中，这样进行git操作的时候就不需要指定特定的用户名，最近在倒腾容器相关，每次启动容器后想进行clone都需要以下两个步骤

  • 通过 ssh-keygen 生成ssh的公钥
  • 把公钥添加到 gitlab 的 ssh keys中

碰到没有安装ssh-keygen的在centos上通过命令安装ssh相关软件。

很烦锁，为了省去这个步骤，通过搜索和自己的实践，有了以下内容

假设

工程的地址

  • ssh协议地址为 `git@git.hostname.com:group-name/project-name.git`
  • https协议地址为 https://git.hostname.com/group-name/project-name.git

使用HTTPS协议

仅指定用户名

然后会提示输入密码，输入正确的密码即可完成clone工作。

指定用户名+密码

这种方式不被推荐，任何形式把密码暴露在空气中都是不安全的。
