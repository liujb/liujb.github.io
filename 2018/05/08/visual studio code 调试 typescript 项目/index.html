visual studio code 调试 typescript 项目

一直有个不好的习惯，调试喜欢用console.log，在小项目逻辑不复杂的情况下这种方式也不是完全不可取吧，但是项目逻辑复杂且依赖的环节多时，这种方式就有点抓瞎啦，刚好今年把团队开发语言从过去的js/es过渡到typescript了，在调试typescript的时候遇到了点问题，特此记录。

遇到的问题

我的项目在开发阶段通过yarn run dev 实际为（gulp default）可以启动，只要通过这种方式启动后，关掉服务，再次启动调试就会出现如图

解决方案，重新 npm run tsc (rm -fr output && tsc) 就OK了，猜测是watch的时候 map 文件错乱了，所以找不到具体的js文件。
