  • Linux-nohup-详解

简介

nohup命令的主要作用就是让普通程序能够在后台执行。
但是像mysql，nginx等程序在设计初就做成了守护进程，所以当我们启动nginx的时候并不需要使用nohup。

nohup语法

  1. nohup默认输出附加到当前目录的nohup.out文件中
  2. 如果当前目录的nohup.out不可写，则输出重定向到$HOME/nohup.out文件中。

常见用法

或者

是否&的区别是，即使终端关闭，或者电脑死机程序是否依然运行。

这种写法把标准输出和标准错误结果都输出到nohup.out文件中，这看似方便，但是当输出很大或者很多程序都往这一个文件写的时候会导致nohup.out文件很大，很排除错误造成很大不便。

所以还有一类用法

或者

这里着重解释一下

  1. >其实就是1>output的缩写，代表的是标准输出
  2. 2> &1就是把标准错误，重定向到标准输出，看得出2代表的就是标准错误，<就是重定向的意思。
  3. 另外，0<表示的是标准输入。

所以这里意思是 将标准输出重定向到myout.file，同时将标准错误重定向到标准输出也就是myout.file，总的来说就是标准输出和标准错误全部都定向到myout.file文件中。

nohup与&的区别

  1. nohup只是让命令忽略SIGHUP命令而已。
  2. &则让命令后台运行，至于该命令是否能脱离终端后继续执行，则要看进程是否为守护进程，如果不是则需要使用nohup来帮忙。
  3. /dev/null是一个无底洞，任何东西都可以重定向到这里，但是无法打开，所以当你不关系输出和错误的时候就可以nohup command > /dev/null 2>&1 &
