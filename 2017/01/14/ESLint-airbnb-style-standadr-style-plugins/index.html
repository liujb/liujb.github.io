针对eslint，airbnb code style 以及 standard style 还有很多的插件都属于混淆不清，特作此调研。拜个山头，以后要用的时候再取。

ESLint

ESLint is an open source project originally created by Nicholas C. Zakas in June 2013. Its goal is to provide a pluggable linting utility for JavaScript.

它的宗旨是给Javascript提供有效的检测工具，跟具体的所谓的xx代码风格无关，它提供了一下预设的规则，或者你也可以自己扩展规则，它不仅仅可以检测js还可以检测css以及HTML等

Eslint-format

根据你的ESLint配置文件自动格式化Javascript代码的Sublime text 3插件。

SublimeLinter-contrib-eslint

This linter plugin for SublimeLinter provides an interface to ESLint. It will be used with files that have the “javascript” syntax.

Airbnb Javascript style

纯粹的提供code style，除了部分针对jshint以及eslint的配置文件，不提供更多的工具，也没有命令行工具，针对各类编辑器是依赖eslint以及相关的插件来实现。

Standard code style

Standard code style提供了Javascript的编写风格，并且提供了对应的command工具以及针对文本编辑器的工具，其中的宗旨是

  1. 不需要配置
  2. 在提交PR之前捕获错误

它不仅仅是风格的呈现，还提供了很多工具，也可以集成到eslint中，通过eslint的工具和插件检测代码风格如有以下插件

  1. eslint-plugin-standard
  2. eslint-config-standard
  3. eslint-plugin-promise

Standard信息

涉及到的其他插件

  1. eslint-friendly-formatter

A simple formatter/reporter for ESLint that’s friendly with Sublime Text and iterm2 “click to open file” functionality`

  1. eslint-loader

eslint loader for webpack

  1. eslint-plugin-html

This ESLint plugin extracts and lints scripts from HTML files.
Supported HTML extensions: .html, .xhtml, .htm, .xml, .vue, .hbs, .mustache, .handelbars, .php, .twig
