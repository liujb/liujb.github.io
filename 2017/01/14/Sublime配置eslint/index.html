报错

当我使用eslint gulpfile.babel.js来检测文件的时候，出现如下报错

Note: eslint --init is intended for setting up and configuring ESLint on a per-project basis and will perform a local installation of ESLint and its plugins in the directory in which it is run. If you prefer using a global installation of ESLint, any plugins used in your configuration must also be installed globally.

解决

意思就是说如果你的eslint是全局安装的，那么相应的插件你也需要全局安装才可以。如果你使用本地初始化的方式，那么所有的依赖包都使用本地的方式安装，反之则使用全局的方式安装

SublimeLinter-contrib-eslint

请详细阅读

Install

  1. 先安装Sublime插件ESLint
  2. 再安装Sublime的Linter插件对应的SublimeLinter-contrib-eslint
  3. 顺便提一嘴，Package controller以及SublimeLinter本身的安装别忘了。

注意：还有个SublimeLinter-contrib-eslint_d先不用管这个，有时间可以去研究，这个是eslint_d的SublimeLinter，至于eslint_d又是什么，其实是eslint的fork的版本，做了一些性能优化的事情，我们暂时还是使用正统的eslint

具体的安装过程不详细讲解，安装完成后，可以看到包列表

注意事项

简单说来，Sublime所有的linter都要依赖SublimeLinter这个插件，在这个插件的基础上再安装各类语言对应的语法检测工具。需要注意SublimeLinter本身的设置

千万注意此时使用的Linter到底是不是ESLint

集成构建工具

看起来好像一切都perfect了，其实不然。编码规范很重要，但是在过去很多团队都无法执行下去，原因是没什么能约束，即使我们努力捯饬出方便的工具，没有人用规范最终只会不了了之。

所以接下来我们要把规范引入到构建工具上，用于在项目构建的时候的必要检测，如果不通过则不予构建成功。目前主流的构建工具主要有gulp和webpack，我们会针对这两个构建工具来集成eslint检测。
