Gulp开发实践备注

Posted on 2016-06-01 | Edited on 2018-08-30

背景交代

前端同学在项目上线的时候总会有将html、js、css、image等资源打包的需求，大部分的做法有：

  1. 简单粗暴，在线压缩一下即可。
  2. 使用 grunt 管理资源.
  3. 国内的同学们很多喜欢使用 fis 来构建
  4. gulp

像grunt和fis我个人理解是已经成型的前端构建的武器。首先他们都很优秀，确实能帮我们解决很多事情。但是作为一个coder要有一颗折腾的心。gulp有很多插件让我们自己来折腾前端构建这个事情。

闲聊扯淡

web开发主要的模式有

  1. 传统的 asp、asp.net、jsp、php（直接echo出html）等面包屑形式。
  2. 采用mvc或mvvm模式来前后端分离比如asp.net mvc，codeigniter，beego 等一大堆。

不管哪种方式，跟前端相关都会涉及到html和static资源。这里面有两种模式

  1. static 和项目中的其他文件在同一个项目下。
  2. static 项目单独出来，跟主项目是两个不同的项目。

此处的html并非特指html文件，泛指各类页面骨架文件(如ejs)，反正最后都会生成html元素。

而前端资源又有两种部署方式，

  1. 一种是js，css，fonts，images等资源直接部署到web server。这种方式有一个显著的坏处就是每一个静态请求都会带上cookie。
  2. 另一种是静态资源直接部署到单独的static server，有条件的可能还加个CDN。

本文的内容适用以上两种方式，即static项目可独立可不独立。

约定和路径

约定
  1. 每个项目下都有一个 src 和 dist 文件夹，分别是源码和构建后的代码
  2. 每个项目下都有package.json和gulpfile.js两个文件。

package.json存储的是依赖包，只需要进行npm install就会在项目下创建node_modules文件夹，且包都在此文件夹下。
gulpfile.js是主文件。

路径
  1. 对static资源（不包括项目骨架html）
    1. minify － 压缩
    2. concat － 合并文件
    3. replace － 有部分js中还引用其他js
    4. …
  2. 对项目骨架文件进行
    1. build html － 比如html中引用了4个js，替换成引用一个合并后的js
    2. minify － 压缩html
    3. replace － 替换路径
    4. 对引用的js和css等做版本hash
    5. …

Git常用命令

Posted on 2016-05-24 | Edited on 2020-03-04

Git 介绍

basic

  1. git clone – 克隆一个项目
  2. git init – 初始化项目
  3. git add – 增加文件
  4. git status – 查看当前的状态
  5. git status -s – 查看当前状态

advanced

  1. git branch – 查看分支
  2. git branch issue01 – 创建新的issue01分支
  3. git checkout issue01 – 切换到issue01分支
  4. git checkout -b issue02 – 新建并切换到issue02分支
  5. git branch -d issue01 – 删除issue01分支
  6. git merge issue01 – 合并issue01分支到当前分支
  7. git reset --hard HEAD –
  8. git push origin branch_name 提交本地分支到远程分支

git tag

  1. git tag – 查看分支
  2. git tag -a 'tagname' -m 'remark' – 创建轻量级tag
  3. git tag -s 'tagname' -m 'remark' – 创建带有签名的tag
  4. git tag -d tagname 删除tag
  5. git push origin tagname – 提交tag到远程
  6. git push origin tag tagname – 提交tag到远程
  7. git push origin --tags – 提交所有的tag
  8. git push origin --delete tagname – 删除远程tag
  9. git push origin --delete tag tagname – 删除远程tag
  10. git fetch origin tag tagname – 获取tag

git branch

  1. git branch – 查看本地分支
  2. git branch -a – 查看所有的本地分支
  3. git branch -d branchname 删除本地分支
  4. git push origin --delete branchname 删除远程分支 git1.7之后
  5. git push origin --delete tag tagname 删除远程tag git1.7之后
  6. git push origin :tagname 删除远程分支 git1.7之前
  7. git push origin :refs/tags/tagname 删除远程tag git1.7之前

git update forked project

git上自己创建了一个项目A，我的同事fork一个B，当我的项目更新的时候，怎么样在他fork的repo上进行相应的更新?

  1. 先把B clone到本地 git clone B_REPOSITORY_URL
  2. 再cd到本地B的目录，把A作为一个remote加到本地的B中（一般命名为upstream）git remote add upstream A_REPOSITORY_URL
  3. pull另一个A的remote（upstream）的相应分支（比如master）就可以 git pull upstream master
  4. 最后push回github的B_REPOSITORY git push origin master

git log

  1. git blame -L 26,35 delete.php

正反代理服务器概念

正向代理和反向代理
百度百科

代理服务器和反向代理，这些概念大家肯定经常听，但是具体有什么异同呢？特意花时间琢磨了，特此记录

正向代理中，proxy和client同属一个LAN，对server透明；
反向代理中，proxy和server同属一个LAN，对client透明。
实际上proxy在两种代理中做的事都是代为收发请求和响应，不过从结构上来看正好左右互换了下，所以把后出现的那种代理方式叫成了反向代理。
