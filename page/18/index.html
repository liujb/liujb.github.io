格式化Nginx配置文件

Posted on 2016-05-02 | Edited on 2020-03-04

经常看到很多服务器上Nginx的配置文件格式上七零八落，本人有轻微的强迫症，看不惯横七竖八的代码，所以开启了Nginx配置文件的格式化摸索之路。

Vim-nginx-syntax-format

在对 Nginx 的配置文件进行编辑的时候，可能出现配置文件缩进很乱的情况，如果手动去修改也挺费事儿的。

好在可以给 vim 增加 nginx.vim 的语法文件来很方便处理配置文件的缩进。使用 vim 的 gg=G 就可以一次把 nginx 的配置文件格式化。

下载文件

点击此处下载

配置文件

  1. 将nginx.vim copy 到 /usr/share/vim/vim73/syntax 文件夹里面，或者~/.vim/syntax 里面
  2. 在 filetype.vim 文件中加入 au BufRead,BufNewFile /usr/local/etc/nginx/* set ft=nginx 其中filetype.vim 可以在 /usr/share/vim/vim73/filetype.vim 也可以在 ~/.vim/filetype.vim

使用命令

最后使用vim的命令 gg=G 即可。

每个程序员都应该有一个Mac电脑用于开发，一方面因为Unix liked，另一方面确实性能好且包括了很多开发需要的环境。

安装程序android-sdk

android-sdk是安卓开发需要的核心组件，说白了就是非他不可。

  1. brew install android-sdk – 这个过程需要翻墙，而且iterm2或者terminate也需要设置代理，然后就是漫长的等待，每每这个时候就羡慕国外程序员，不用翻墙，不用翻译，效率肯定快不少。

过程中遇到的问题

1. 第一次执行react-native run-android的时候会download https://services.gradle.org/distributions/gradle-2.4-all.zip 然后网速不快，会一直在.......中，这个时候有两种方式

  1. 耐心等待，最终还是会成功的。
  2. 在ReactNativeDemo中grep -r 'gradle-2.4-all.zip' . 找到有一个配置文件，更改配置文件的只，要么你通过某种技巧将gradle-2.4-all.zip放到本地，然后更改目录，如果提示不存在，注意看路径。

2. 编译失败，提示 Gradle version 2.2 is required. Current version is 2.10

这是因为，通过brew install gradle安装的版本是2.10而，在编译过程需要使用的是2.2(其实是2.4，因为配置文件中写的就是2.4)，这个时候就是实用Q1那个，安装2.4版本的gradle即可。

3. 编译失败，提示

这是由于模拟器没有启动，解决方法还是启动模拟器或者连接上物理安卓设备（要打开USB调试），然后用adb devices查看设备列表，有设备列表，在执行react-native run-android即可。

另外这个问题，还有两种可能的情况

  1. 就是Android SDK Build-Tools跟代码中指定的编译器版本不一致。
  2. https://github.com/facebook/react-native/issues/2711 这里提到，没有奖模拟器的Use Host GPU 也会导致该问题。

Web扫描之media调研

Posted on 2016-04-21 | Edited on 2020-03-04

背景

项目中需要用到 H5 实现扫描用户证件，然后识别出证件上的具体信息，填充到页面上指定的文本框内。

解决方案

  1. JS本地扫描，JS本地识别，DEMO，有人说实现不了，我只是需要告诉他纯实现还是可以的，但是兼容效果和准度确实是大问题。
  2. JS通过用户授权使用相机自动扫描并拍照，而后自动上传图片到server，然后在Server上利用python或者c++等实现识别。
  3. 调起用户相机或者用户或图库，然后上传图片到Server，而后实现扫描。

以上方案依赖的技术点主要有

  1. HTML5的API getUserMediaedia 。
  2. JS实现OCR功能的 OCRAD
  3. 移动端选择图片上传等功能。

用户体验（不包括识别准度）: 方案1 > 方案2 > 方案3
识别准度: 方案3 == 方案2 > 方案1

调研结论

实现此功能目前还是只能通过方案3，即

  1. 间接调起用户的拍照功能或者相册，用户选择图片
  2. 将图片上传到server端，server端识别后再返回。

来完成，虽体验不及前两类好，但是在现有的条件下这是H5唯一能做的。

若在唯一的环境比如某APP中，最好的体验应该是Native app提供这类功能，然后H5通过JS和Native通讯，共同做扫描，这个事情需要Native发版支持。

配置系统简要设计

Posted on 2016-04-20 | Edited on 2020-03-04

项目背景

日常有很多更改配置类的需求，比如

  1. 功能开关的开闭。
  2. 文案的更改
  3. 配置的更改

等需求目前都是RD更改代码，而后发上线单，然后上线，整个过程很low且效率低下，鉴于此，开发配置管理工具，借此释放RD的劳动力，让运营和PM也能参与配置更改。

项目目标

  1. 短线目前先释放团队部分RD的劳动力，长线希望能在业务团队推广使用。
  2. 实现部分配置自动化，确保配置一致性。

项目流程

流程是：web界面上编辑－》生成文件－》同步到server

配置管理主要有以下几类方法

  1. rsync方式将文件同步
  2. crontab+pull方式
  3. redis+mysql方式
  4. configman同步。

其中

  1. rsync方式一致性没办法确保。
  2. crontab+pull方式不支持小流量和分组上线
  3. redis+mysql方式对既有代码兼容太差，且实现方式low
  4. configmanaaa方式
    • 利用git做版本管理方便回滚
    • 有一致性保障机制
    • 发现文件不一致有报警机制
    • 支持小流量和分组

对比下来如果configman真的像作者描述的那样，那基本上满足需求。

多种配置文件支持

需要支持多种配置文件实现，前期可以只实现ci-php配置

  1. php
  2. json
  3. yaml

采用类似一种委托的机制，把生成文件的过程单独切分出来，实现一个适配器function，这样后续增加新的文件类型就不怎么需要更改既有代码，同理 语法检测 等对文件本身有操作的都应该采用该机制。

生成配置文件

  1. 根据使用者需要，支持多种配置文件生成，前期只需要支持ci-php形式的config文件。
  2. 生成文件的时候需要对文件锁定，避免并发下同时修改同一文件导致冲突。

语法检测功能

对不同的配置文件，都需要进行语法检测，其实这个过程可以控制在git上。

下载生成的配置文件

提交git功能

小流量功能

通过configman提供的服务，需要支持小流量分发上线。

  1. 编辑完配置后需要将文件通过命令提交到git，
  2. 通知configman服务小流量分发配置。

分组上线

通过configman提供的服务，支持分发到指定一台或多台机器。

  1. 前提是必须先小流量，因为小流量时才会保存文件并提交到git
