背景

通过配置DNS欺骗，达到preview环境验证服务的case。直白的说就是浏览器从发起对域名1的请求，而通过DNS欺骗的手段，让请求真实到达域名2所在机器。

假设域名1和域名2为：

  1. gsb.ins.xxx.com
  2. gsb.preview.xxx.com

这种方式还可以欺骗浏览器，在做微信开发的时候经常使用。

现象

代码中访问的是gsb.ins.xxx.com，但是需要将实际请求落到gsb.preview.xxx.com，一直以来都是利用Charles的DNSSpoofing方式进行，配置如图：

配置成功效果如图

可以清晰得看到Remote Address 为我们的目标地址。正常来讲这就可以了，况且在过去很长时间我都是这么玩的。但是实际效果并非如此（通过比对两个服务接口的返回不同）。经过排查得到两个域名对应的IP地址其实是同一个。

  1. gsb.ins.xxx.com ＝》123.151.179.1
  2. gsb.preview.xxx.com ＝》123.151.179.1

所以这样子通过DNS-Spoofing还是访问的同一个IP。因为上游机器在辨别两个域名的时候，是通过nginx rewrite的方式进行的，所以通过域名2请求的服务还是落到了域名1所在的机器，故问题得到查明。

原则上两个域名应对应两个IP

FastCGI

  1. FastCGI，也称为FCGI，是一个外部服务与Web服务器的接口协议。FastCGI是早期的通用网关接口(CGI)的增强版本。 FastCGI致力于减少网页服务器与CGI程序之间的开销，从而使服务器可以同时处理更多的网页请求。

  2. Nginx为了许多外部工具和服务而引入FastCGI技术，例如Perl, PHP 和 Python。所以，在FastCGI服务启动之前，它们都是不能使用的。

  3. 有不同的方式可以启动PHP FastCGI服务,我们将介绍php-fpm.这也是推荐的方案.

Git操作之Fetch和Pull的区别

Posted on 2016-07-16 Edited on 2018-09-01

Git中从远程的分支获取最新的版本到本地有这样2个命令，fetch 和 pull，对于这两者之间的区别并不熟悉，特此记录。

fetch

相当于是从远程获取最新版本到本地，不会自动merge。

以上命令的含义

  1. 首先从远程的origin的master主分支下载最新的版本到origin/master分支上。
  2. 然后比较本地的master分支和origin/master分支的差别。
  3. 最后本地进行合并。

上述过程其实可以用以下更清晰的方式来进行

从远程获取最新的版本到本地的test分支上之后再进行比较合并

pull

从远程获取最新版本并merge到本地。

命令其实相当于git fetch 和 git merge

SVN命令记录

Posted on 2016-07-16 Edited on 2018-08-30

SVN命令

svn命令详解

基础

  1. svn list svn://svndomain/server/static/ – show list
  2. svn co svn://svndomain/server/trunk/api/v2/views – checkout
  3. svn add xx – add files or folders
  4. svn import $DIR $URL -m “Something your want to mark.” – import local folder to svn and commit to remote
  5. svn rm file – remove
  6. svn mv old_file new_file – move or rename
  7. svn up – update
  8. svn ci filename -m "checkin note” – checkin
  9. svn st -v/-u – show status
  10. svn log filename – show logs
  11. svn cat -r 10528 home.html – show history
  12. svn delete filename – delete
  13. svn delete –force over-there – force to delete

进阶

  1. svn export svn://svndomain/server/trunk/api/v2/views – use for release project, without .svn folder
  2. svn diff -r 12302:12304 home.html – diff two files
  3. svn diff filename – show difference from the working file and the latest version
  4. svn diff get_hb_list.tpl |vim -R - – use vim to diff two files
  5. svn diff filename -r1234:2345[|vim -R -] – use vim to diff two files
  6. svn up –r 200 file.c – 将本地的file.c还原为200版本，【本地是拿下来了，版本库端并没有改变】
  7. svn mkdir newdir
  8. svn mkdir newdir -m "Making a new dir" svn://svndomain/server/
  9. svn resolve –accept working compressed，svn revert compressed – 解决目录compressed树冲突
  10. svn revert filename
  11. ci的时候出现 xx.html is out of date – 解决方法就是先update一下，然后再ci

问题列表

冲突的解决方法：

  1. 先将.mine文件拷贝到你自己的文件
  2. 先跟test.html跟低版本的比较
  3. 再将test.html跟高版本的比较
  4. 解决完冲突之后再diff一下
  5. 然后设置svn resolved test.html
  6. 然后ci

Posted on 2016-07-16 Edited on 2020-03-04

写在前面

楼主是个喜欢折腾的同学，之前也在sublime text中配置过各类语言的linter，jshint，csslint等。来来回回折腾几回后，也没有完全用上，或者说没弄明白吧，再次来配置的时候还是不能一蹴而就，故作记录。

本文假设

  1. 你使用的sublime text 3，因为sublime text 2的sublimelinter已经不维护了。
  2. 最好使用的是unix like环境。
  3. 已具备翻墙，肉翻的人直接忽略。

环境准备

  1. sublime text 3且安装了package control，还没配置的请点击此处
  2. nodejs环境好使，因为jshint和csslint依赖node做check引擎，若不需要直接忽略吧。
  3. 很多插件都依赖诸如，python，ruby，java等基础环境，建议都装上，然后把各自对应的执行文件都加入到PATH里去吧

理论知识

  1. 用于高亮提示用户编写的代码中存在的不规范和错误的写法，支持 JavaScript、CSS、HTML、Java、PHP、Python、Ruby 等十多种开发语言
  2. 可以理解为sublimelinter是一个代码检测容器，可以在其上边配置数十类语言的监测，但是真正的检测引擎各类语言都有自己对应的引擎

安装步骤

  1. command+shift+p或者windows上ctrl+shift+p调出package control，输入install package

  1. 输入sublimelinter
