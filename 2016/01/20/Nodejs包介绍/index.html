Nodejs包介绍

Posted on 2016-01-20 Edited on 2018-09-03

Introduce

npm是node package management的缩写，就是nodejs领域内的包管理软件。通过npm可以安装第三方nodejs包，或者发布自己的nodejs包。安装完Nodejs之后，就自带了npm。通过npm --version可以查看npm的版本。

Nodejs-package

  • package.json（有比较严格的规范）
  • bin
  • lib
  • doc
  • test

Nodejs中对包没有那么严格，只需要有package.json文件就行了

Using

npm支持的命令如下

常用的命令就是npm install，分两种模式

  1. 全局模式：npm install packageName -g 这部分的包安装在/usr/local/lib/node_modules或者/usr/lib/node_modules
  2. 本地模式：npm install packageName 这部分包安装在项目内的node_modules

如果npm install后边不接包名，则会默认安装package.json内的包到本地node_modules

require的时候会先搜寻本地的node_modules，而后搜索全局的node_modules，如果都没有的话就会报错。

package and modules

  1. 把文件夹封装成一个模块，就是包！
  2. 模块和文件对应，一个文件就是一个模块，这个文件可以是js代码，json或者C/C++的扩展
  3. 两个对象 exports和require，用来公开接口和从外部获取模块的接口，即所获的exports对象
  4. 有时候想把对象封装到模块中，使用module.export = myObj;来取代exports.myObj = myObj;
  5. 如果要把包当作工程的一部分，通过本地获取
  6. 如果要在命令行下使用，则使用全局安装（不能通过require来获取）
  7. 可以通过npm link命令打破全局安装的包不能通过require来获取的限制。

npm的包管理器非常方便，只需要package.json文件就能管理好工程中所需要的包。
