ReactNative快速入门

Posted on 2016-05-02 Edited on 2020-03-04

本文主要介绍在Mac上搭建ReactNative环境进行ReactNative技术尝新，看起来好像比较简单，对于未曾做过Native开发的同学还是遇到了不少的问题，故记之。主要参考文章如下

Facebook-ReactNative-Getting-Started
Facebook-Android-Env-Setup

环境准备

每个程序员都应该有一个Mac电脑用于开发，一方面因为Unix liked，另一方面确实性能好且包括了很多开发需要的环境。

目前价格可以接受，可能刚毕业的学生无法承受，工作过一段时间的朋友记得一定要买Mac，不使用Mac的程序员是不完整的。

主要的环境有

  1. Mac OSX 本文都是在Mac下进行的
  2. Homebrew Mac上进行包管理的不二选择，还可以使用brew cask进行APP安装和管理。
  3. Nodejs 因为ReactNative是主要使用Javascript语言进行开发，nodejs是必备的brew install node进行安装。
  4. npm
  5. brew install watchman
  6. brew install flow
  7. brew install git
  8. JAVA环境，并且设置好JAVA_HOME，CLASSPATH，PATH

iOS设置

从App Store安装Xcode最新版本，或者从这里连接到App Store安装即可。

Android设置

安装程序android-sdk

android-sdk是安卓开发需要的核心组件，说白了就是非他不可。

  1. brew install android-sdk – 这个过程需要翻墙，而且iterm2或者terminate也需要设置代理，然后就是漫长的等待，每每这个时候就羡慕国外程序员，不用翻墙，不用翻译，效率肯定快不少。
  2. 设置export ANDROID_HOME=/usr/local/opt/android-sdk – 安装完后，会有一个快捷方式指向/usr/local/opt/android-sdk，我们只需要在~/.zshrc或者~/.bashrc下设置 ADNROID_HOME即可。

安装成功的标志是可以执行android命令，也可以更详细的了解文件安装情况

  1. which android 出现 /usr/local/bin/android 然后 ll /usr/local/bin/android 出现 lrwxr-xr-x 1 liujb admin 42B Jan 4 22:58 /usr/local/bin/android -> ../Cellar/android-sdk/24.4.1_1/bin/android
  2. echo $ANDROID_HOME 会出现/usr/local/opt/android-sdk，再 ll /usr/local/opt/android-sdk发现指向 lrwxr-xr-x 1 liujb admin 30B Jan 4 22:58 /usr/local/opt/android-sdk -> ../Cellar/android-sdk/24.4.1_1

其中有几个注意项目

  1. Android x.x.x (API xx) 这部分并不需要每个版本都安装，但是安装的那个版本最好对应上Intel开头的那两项

安装Genymotion模拟器

Genymotion是一个安卓模拟器软件，对个人账户免费，所以使用前需要先注册。该模拟器软件比sdk自带的模拟器软件性能好很多，运行速度快，主要原因是其是基于虚拟机的。所以安装Genymontion之前必须先安装VirtualBox，没错，就是那个虚拟机软件。使用brew cask install genymotion安装即可。

过程中遇到的问题

1. 第一次执行react-native run-android的时候会download https://services.gradle.org/distributions/gradle-2.4-all.zip 然后网速不快，会一直在.......中，这个时候有两种方式

  1. 耐心等待，最终还是会成功的。
  2. 在ReactNativeDemo中grep -r 'gradle-2.4-all.zip' . 找到有一个配置文件，更改配置文件的只，要么你通过某种技巧将gradle-2.4-all.zip放到本地，然后更改目录，如果提示不存在，注意看路径。

2. 编译失败，提示 Gradle version 2.2 is required. Current version is 2.10

这是因为，通过brew install gradle安装的版本是2.10而，在编译过程需要使用的是2.2(其实是2.4，因为配置文件中写的就是2.4)，这个时候就是实用Q1那个，安装2.4版本的gradle即可。

3. 编译失败，提示

这是由于模拟器没有启动，解决方法还是启动模拟器或者连接上物理安卓设备（要打开USB调试），然后用adb devices查看设备列表，有设备列表，在执行react-native run-android即可。

另外这个问题，还有两种可能的情况

  1. 就是Android SDK Build-Tools跟代码中指定的编译器版本不一致。
  2. https://github.com/facebook/react-native/issues/2711 这里提到，没有奖模拟器的Use Host GPU 也会导致该问题。
