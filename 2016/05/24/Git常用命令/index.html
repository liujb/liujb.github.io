Git常用命令

Posted on 2016-05-24 Edited on 2020-03-04

Git 介绍

basic

  1. git clone – 克隆一个项目
  2. git init – 初始化项目
  3. git add – 增加文件
  4. git status – 查看当前的状态
  5. git status -s – 查看当前状态

advanced

  1. git branch – 查看分支
  2. git branch issue01 – 创建新的issue01分支
  3. git checkout issue01 – 切换到issue01分支
  4. git checkout -b issue02 – 新建并切换到issue02分支
  5. git branch -d issue01 – 删除issue01分支
  6. git merge issue01 – 合并issue01分支到当前分支
  7. git reset --hard HEAD –
  8. git push origin branch_name 提交本地分支到远程分支

git tag

  1. git tag – 查看分支
  2. git tag -a 'tagname' -m 'remark' – 创建轻量级tag
  3. git tag -s 'tagname' -m 'remark' – 创建带有签名的tag
  4. git tag -d tagname 删除tag
  5. git push origin tagname – 提交tag到远程
  6. git push origin tag tagname – 提交tag到远程
  7. git push origin --tags – 提交所有的tag
  8. git push origin --delete tagname – 删除远程tag
  9. git push origin --delete tag tagname – 删除远程tag
  10. git fetch origin tag tagname – 获取tag

git branch

  1. git branch – 查看本地分支
  2. git branch -a – 查看所有的本地分支
  3. git branch -d branchname 删除本地分支
  4. git push origin --delete branchname 删除远程分支 git1.7之后
  5. git push origin --delete tag tagname 删除远程tag git1.7之后
  6. git push origin :tagname 删除远程分支 git1.7之前
  7. git push origin :refs/tags/tagname 删除远程tag git1.7之前

git update forked project

git上自己创建了一个项目A，我的同事fork一个B，当我的项目更新的时候，怎么样在他fork的repo上进行相应的更新?

  1. 先把B clone到本地 git clone B_REPOSITORY_URL
  2. 再cd到本地B的目录，把A作为一个remote加到本地的B中（一般命名为upstream）git remote add upstream A_REPOSITORY_URL
  3. pull另一个A的remote（upstream）的相应分支（比如master）就可以 git pull upstream master
  4. 最后push回github的B_REPOSITORY git push origin master

git log

  1. git blame -L 26,35 delete.php
