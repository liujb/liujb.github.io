Gulp开发实践备注

Posted on 2016-06-01 Edited on 2018-08-30

背景交代

前端同学在项目上线的时候总会有将html、js、css、image等资源打包的需求，大部分的做法有：

  1. 简单粗暴，在线压缩一下即可。
  2. 使用 grunt 管理资源.
  3. 国内的同学们很多喜欢使用 fis 来构建
  4. gulp

像grunt和fis我个人理解是已经成型的前端构建的武器。首先他们都很优秀，确实能帮我们解决很多事情。但是作为一个coder要有一颗折腾的心。gulp有很多插件让我们自己来折腾前端构建这个事情。

闲聊扯淡

web开发主要的模式有

  1. 传统的 asp、asp.net、jsp、php（直接echo出html）等面包屑形式。
  2. 采用mvc或mvvm模式来前后端分离比如asp.net mvc，codeigniter，beego 等一大堆。

不管哪种方式，跟前端相关都会涉及到html和static资源。这里面有两种模式

  1. static 和项目中的其他文件在同一个项目下。
  2. static 项目单独出来，跟主项目是两个不同的项目。

此处的html并非特指html文件，泛指各类页面骨架文件(如ejs)，反正最后都会生成html元素。

而前端资源又有两种部署方式，

  1. 一种是js，css，fonts，images等资源直接部署到web server。这种方式有一个显著的坏处就是每一个静态请求都会带上cookie。
  2. 另一种是静态资源直接部署到单独的static server，有条件的可能还加个CDN。

本文的内容适用以上两种方式，即static项目可独立可不独立。

约定和路径

约定
  1. 每个项目下都有一个 src 和 dist 文件夹，分别是源码和构建后的代码
  2. 每个项目下都有package.json和gulpfile.js两个文件。

package.json存储的是依赖包，只需要进行npm install就会在项目下创建node_modules文件夹，且包都在此文件夹下。
gulpfile.js是主文件。

路径
  1. 对static资源（不包括项目骨架html）
    1. minify － 压缩
    2. concat － 合并文件
    3. replace － 有部分js中还引用其他js
    4. …
  2. 对项目骨架文件进行
    1. build html － 比如html中引用了4个js，替换成引用一个合并后的js
    2. minify － 压缩html
    3. replace － 替换路径
    4. 对引用的js和css等做版本hash
    5. …
